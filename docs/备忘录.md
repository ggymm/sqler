### BUG 修复

进入 Bug 修复模式：

1. 立即行动，减少解释：直接定位和修复，不要过度规划
2. 并行探索：同时读取多个可能相关的文件，快速建立上下文
3. 快速迭代：先尝试修复，再验证，不要等待完美方案
4. 使用错误信息：从错误堆栈/日志开始，直接定位到问题代码
5. 及时转向：如果同一方向尝试 2-3 次仍失败，立即换思路，不要钻牛角尖
6. 简洁报告：只说明发现了什么问题、怎么修的、如何验证

### 降智问题

1. 避免纠正AI，出错了直接重头再来
2. 实现功能时，尽量保证在两次对话完成
3. 如果无法保证第 2 点，要使用计划模式，重新拆分任务
